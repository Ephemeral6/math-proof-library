# Condition R00 — Relation only (graph_connectivity)

## 数据分析

`detailed_comparison` 给出每个 case 的 `deleted_edge_was_bridge` 标签和 `n_bridges_pre/post`；
`structural_comparison` 给出 `components_pre/post` 和 `articulation_points_pre/post`。

跨 100 个 case 检查**完美的 1-1 对应关系**：
| `deleted_edge_was_bridge` | `connectivity_lost` | count |
|---|---|---|
| true  | true  | 17 |
| false | false | 83 |
| true  | false | 0  |
| false | true  | 0  |

进一步从 `components_pre/post` 可以看到：
- 当 `connectivity_lost=true` 时，`components_post` 总是把原 component 分成恰好两块——`u` 在一块、`v` 在另一块。
- `articulation_points` 的变化：删 bridge 后，被分裂出的孤立顶点（如 deg=1 的端点）从关节点列表中消失；非孤立分裂点的关节点状态也可能改变。

## 论证尝试

**定理**：设 G 连通，e=(u,v)∈E(G)。则 G−e 不连通 ⇔ e 是桥（在 G 中删除 e 后 u, v 不再属于同一连通分量）。

**证明**：
- (⇐) 若 e 是桥，按定义 G−e 中 u, v 不连通；由于 G 连通，u 所在分量包含原 G\{e} 中所有从 u 可达的点，v 不在其中，故 G−e 至少有 2 个分量。
- (⇒) 设 G−e 不连通。G 连通意味着 G−e 仍包含 G 的所有顶点（删边不删点）。G−e 不连通 ⇒ ∃ 顶点对在 G−e 中不可达。取 u 所在分量 C_u 和它的补 C_v；既然 G 连通而 G−e 把它劈开，唯一被破坏的连接就是 e=(u,v)，故 u∈C_u, v∈C_v，e 是桥。

数据完全支撑这个等价：100/100 case 的 `deleted_edge_was_bridge ⇔ connectivity_lost` 都成立，且 `components_post` 中的 (C_u, C_v) split 在每个 connectivity-lost case 中可见。

更进一步的观察（数据支持但不必要）：删除非桥后 `n_bridges_post` 可能上升（如 case `R00_n8-t0`: 1→3），这是因为非桥的删除可能让原本属于环的某些边变成桥（环被破坏）。

## 自评

**评分：PROOF（4）**

理由：
- 定理陈述清晰、双向；
- 证明双向都给出，仅依赖"连通=任一顶点对存在路径"和"删边不删点"两个标准事实；
- 数据完美支撑（100/100 一致），且 components 的 split 结构在数据中肉眼可见，是机械验证级别的证据。

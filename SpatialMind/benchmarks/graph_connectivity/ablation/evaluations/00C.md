# Condition 00C — Counterfactual only (graph_connectivity)

## 数据分析

100 个 (G, e) pair 的 `summary_delta`：17 case `n_components_a=+1`（连通性丢失），83 case `=0`。

`counterfactual` 块包含 2 个 critical case：

1. **boundary_relaxation**: 同一个图 G 上对比"删非桥 (1,2)"与"删桥 (4,5)"。
   - 删 (1,2): `n_components_after=1`, `connectivity_lost=false`。
   - 删 (4,5): `n_components_after=2`, `connectivity_lost=true`。
   - 解释（CF 自带）："the bridge property (no alternative path between endpoints) is what determines whether connectivity is preserved under edge deletion."

2. **operation_perturbation**: 删边 vs 加边的方向不对称。
   - 删 (0,1): `components_change=+1`。
   - 加 (0,5): `components_change=0`（这里加边没合并，因为图原本就连通）。
   - 揭示：删边只能让 components 不变或 +1，加边只能让 components 不变或 -1。

## 论证尝试

**结合 CF + summary 我能构造的论证**：

CF (1) 提出假设：连通性丢失 ⇔ 删的是 bridge（"无替代路径"的边）。
summary 显示 17/100 case 连通性丢失。**如果**这 17 case 删的都是 bridge、其余 83 case 删的都是非 bridge，则数据完全支持假设。

但我的局限：summary 不告诉我每个 case 删的边是不是 bridge——CF 只在 1 个特定图上展示了 contrast，没有覆盖那 100 个 case。

**我能写出的论证**：
> 由 CF (1)：在某个示例图中，"删 bridge"⇒"connectivity 丢失"，"删非 bridge"⇒"connectivity 保持"。
> 由 CF 的解释文字："bridge = u, v 之间没有其他路径"。
> 用图论标准事实：若 (u, v) 是图中唯一的 u-v 路径，则去掉它就让 u, v 不可达，故 G−e 不连通。反之若有第二条路径，删 e 后这条路径仍在，G−e 连通。
> 因此连通性丢失 ⇔ 被删的边没有替代路径 ⇔ 是桥。
> summary 的 17/83 split 与此假设一致（具体一一对应需要 R 或 T 才能验证）。

CF (2) 是另一个独立观察：操作的方向性。这暗示"connectivity loss"是删边特有的现象，加边不能引起，所以问题的*本质*是"边的存在为连通性提供保障"，bridge 是这种保障是否唯一的判据。

## 自评

**评分：ARGUMENT（3）**

理由：
- CF 的 explanation 文本和 boundary contrast 给出了 bridge 概念和正确的等价方向；
- 我能调用图论标准事实把 CF 的两个 case 推广到 universal claim；
- 但 100 个 case 上的具体验证只能对总数做检验（17 vs 83），不能在 case 层面对 bridge 标签做检验——CF 只在 1 个 demo 图上展示对比；
- 因此论证有完整的 *if-then* 结构，但缺少 100-case 数据的 case-level 支撑。

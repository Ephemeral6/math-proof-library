# Condition 0T0 — Transform only (graph_connectivity)

## 数据分析

每个 case 的 `transform_trace` 包含：
- `delta.is_bridge`: bool 标签。
- `delta.connectivity_lost`: bool 标签。
- `delta.components_change`: 0 or +1。
- `region_affected.deleted_edge`, `endpoint_degrees_before/after`。

跨 100 个 case，三元组 `(is_bridge, connectivity_lost, components_change)` 的取值统计：
- (true, true, +1) × 17
- (false, false, 0) × 83
- 其他组合 × 0

再加一个观察：端点度数 `endpoint_degrees_before` 中**至少**一个为 1（即叶子）的 case 全部是 bridge（删叶子边肯定是桥）；但 `endpoint_degrees_before` 都≥2 的 case 也有部分是 bridge——所以"叶子边"是 bridge 的充分条件，不是必要条件。

## 论证尝试

数据给出的是**操作层标签**：`is_bridge` 是 trace 内部计算出的 flag，等价于 `components_change == +1`。

**经验等价**：在 100 个 case 上，
> `is_bridge` ⇔ `connectivity_lost` ⇔ `components_change == +1`。

第三个等价是 trace 的恒等式（删边后分量数变化 = 1 当且仅当 connectivity 从 true 变 false，前提是图原本连通——所有 100 case 都满足）。前两个标签的等价是数据观察而非定义；trace 没告诉我"bridge"的图论定义是什么。

我可以基于这些 trace 内部一致性写出**条件命题**：
> 若一条边 e 满足 "G−e 与 G 的连通分量数差 1"，则 e 是 bridge（操作定义意义上的）。
> 反之亦然。

但这不是严格意义上的"证明 bridge ⇔ 连通性丢失"——它是**重新表述**：bridge 定义为"删它后分量增加 1 的边"，于是 connectivity_lost 等价是 trivially true。

我**不能**从 trace 数据本身证明"为什么 bridge 是这样"——例如不能证明"u, v 在 G−e 中不连通 ⇒ G 中除 e 外没有 u→v 的其他路径"——因为 trace 不暴露图的完整边列表，只显示端点度数变化。

## 自评

**评分：ARGUMENT（3）**

理由：
- 100 case 上的完美 1-1 对应是强经验事实；
- 我可以陈述等价并指出它在数据中处处成立；
- 但"bridge"在 trace 里只是个标签，缺少图论意义上的*定义*（用 R 或 CF 才能给出）；
- 因此我能写出 *operational* equivalence 的论证，但写不出 *structural* proof。

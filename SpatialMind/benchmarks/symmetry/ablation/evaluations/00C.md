# Condition 00C — counterfactual only (symmetry_combinatorics)

## 数据分析
summary 之外多了 3 个反事实 case：

**CF-1 boundary_relaxation（Z_6 → D_6）**：
- 原条件：`group=Z_6, group_order=6, total_orbits=130`  
- 修改后：`group=D_6, group_order=12, total_orbits=92`  
- 在代表 pair 上 `same_orbit` 不变（True → True）  
- `condition_is_critical = False`：群放大没改变这对 (a, b) 的 same_orbit。
- 但 `total_orbits: 130 → 92`（差 -38）：**有 38 对 Z_6 不同轨道的着色在 D_6 下被合并**。

**CF-2 condition_removal（Z_6 → {e}）**：
- 原条件：`group=Z_6, total_orbits=130`  
- 修改后：`group={e}, group_order=1, total_orbits=729`  
- 同 pair 上 `same_orbit: True → False`  
- `condition_is_critical = True`：群是关键条件，无群则 same_orbit 退化为 a == b。
- 轨道数 130 → 729（每个 coloring 一个轨道）。

**CF-3 operation_perturbation（非群置换 swap(0,1)）**：
- 原：用群元素 g（element_index=1）→ σ_a 与 a 同轨道（True）  
- 修改：用非群置换 swap(0,1) → 结果 `[2,0,1,2,0,0]`，与 a 不同轨道（False）  
- `condition_is_critical = True`：非群置换破坏 orbit equivalence。

## 论证尝试

CF 给出三个关键信号：

**(C1) 群 = 等价关系的来源**（CF-2）：  
G = {e} 时同轨道退化为 ==。所以「轨道」概念依赖 G 的非平凡结构。

**(C2) 群结构本身决定轨道数**（CF-1）：  
Z_6 → D_6 把 130 个轨道压成 92 个（差 38）。这说明轨道数不是由 |G| 单独决定的（Z_6 的 |G|=6, D_6 的 |G|=12），而是由 G 的具体作用决定的。  
注意：(1/12)·(Σ_{g∈D_6} |Fix(g)|) = 92，所以这是 D_6 自己的 Burnside 数。

**(C3) 必须用群元素**（CF-3）：  
swap(0,1) 这个非群置换不属于 Z_6，所以 swap·a 与 a 在 Z_6 作用下不一定同轨道。这反过来证明：「群作用保持轨道」依赖于「g ∈ G」这个前提。

但 CF 没有给我：  
- `stabilizer_size`、`fixed_point_counts`、`burnside_count` 等正面数据；  
- 任何一个 (a, b) pair 的具体 same_orbit 数值（只有 1 个代表 pair）；  
- 顶点循环结构。

我只能从 CF 反推：

**轨道数 = (1/|G|) Σ_g |Fix(g)|** 这个等式，CF 上没有任何直接证据。CF-1 给出「D_6 下有 92 个轨道」、CF-2 给出「{e} 下有 729 个轨道」，但 (1/12)·X = 92 和 (1/1)·729 = 729 这两个等式背后是 Burnside，不是 CF 数据本身能验证的。

我能做的论证：  
- CF 表明等价关系**完全由 G 决定**（C1+C3）。  
- CF 表明 G 不同则轨道数不同（C2 量化）。  
- 但 CF 不告诉我「为什么轨道数恰为 130」、不告诉我 orbit-stabilizer 是怎么回事。

## 自评
**评分：PATTERN（2）**

理由：  
- CF 给出三条关键存在性陈述（G 是必要条件、群放大压缩轨道、非群置换破坏等价）。  
- 这些都是 *负面信号* —— 说明 G 必不可少，但不构成 Burnside 引理本身的证明。  
- 没有 stabilizer 数据、没有 Fix(g) 数据，无法搭起双重计数论证。  
- 比 000 多了「群是 critical condition」的明确信号，但仍触不到证明骨架。

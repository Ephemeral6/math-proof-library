# Condition 00C — Counterfactual only (boundary_interior)

## 数据分析

87 个 case 的 `summary_delta`（多数全 0；scale_non_uniform 让 A、B、I 同时变）。
3 个 critical CF：

1. **boundary_relaxation**（lattice → 非 lattice）：
   - 正例 rectangle_4x3 是 lattice：A=12, B=14, I=6, pick_holds=True。
   - 反例把一个顶点偏移 0.5：A=11.25, B=-1（undefined）, I=-1（undefined）, pick_holds=False。
   - 解释："Pick's theorem requires lattice (integer) vertices. Moving one vertex by 0.5 makes B and I … ambiguous — Pick's formula no longer applies."

2. **operation_perturbation**（unimodular shear vs non-uniform scale）：
   - 正例 shear det=1：A 不变，B、I 不变。
   - 反例 scale [2,1] det=2：A 翻倍，B、I 同步增加但*仍满足 Pick*（A_new = I_new + B_new/2 − 1）。
   - 解释："Linear maps with |det|=1 (unimodular) preserve both area AND lattice structure. Maps with |det|≠1 still send lattice polygons to lattice polygons (if entries are integers), so Pick continues to apply, but the area scales by the determinant."

3. **condition_removal**（简单 → 自交）：
   - 正例简单四边形：Pick 成立。
   - 反例 bowtie：is_simple=False, pick_holds=False。
   - 解释："Pick's theorem assumes the polygon is *simple*. For a self-intersecting polygon, 'interior' is ambiguous … the I count is meaningless."

## 论证尝试

**CF 给出 Pick 定理的成立条件**：
1. 多边形必须是 lattice（顶点整数坐标）。
2. 多边形必须是 simple（边不自交）。

**CF 给出操作不变性的精确判据**：
- 整数 unimodular 线性变换（det=±1）保 A、B、I。
- 整数非 unimodular 线性变换（det≠±1）按 |det| 缩放 A，但仍是 lattice 多边形 ⇒ Pick 仍适用 ⇒ I + B/2 − 1 跟着按 |det| 缩放。

**论证（结合 summary + CF）**：

由 CF (1) 和 CF (3) 的 explanation：Pick 公式的 *应用域* 是 {lattice 简单多边形}。在这个域内，A、B、I 都是良定义的整数（A 半整数，因为单位三角形面积 1/2）。

由 CF (2) 的 explanation：在应用域内，变换分两类：
- |det|=1（保面积）：(A, B, I) 三者不变（unimodular 在 lattice 上是双射）。
- |det|>1（不保面积）：A 按 |det| 缩放，B 和 I 也协同缩放（Pick 仍成立）。

summary_delta 数据给出**经验支持**：
- 71/87 case `area_preserved=True`（即 area_a 在 summary 中 = 0）：包括 translate、shear、add_vertex、identity_compare。这些都是面积不变的操作。
- 16/87 case `area_preserved=False`：scale_non_uniform。跟 CF (2) 的 det≠1 论断一致。
- 87/87 case `pick_holds_a=0` 即 `pick_holds_b=0` 即两侧都成立：Pick 在所有变换下持续适用——这与 CF "lattice 简单 ⇒ Pick 成立" 一致（因为我们的所有变换都保 lattice 简单性）。

**Pick 公式本身（A = I + B/2 − 1）**：
- summary 没给 A、B、I 的具体值——只看 delta。所以*不能从 0TC 验证 Pick 公式*。
- 但 CF 的 original_result 字段给了 A=12, B=14, I=6（rectangle_4x3）：12 = 6 + 7 − 1 ✓。这是单点验证。
- CF (2) 的 original/counterfactual_result 给了 shear 后 (12, 14, 6) 和 scale 后 (24, 22, 14)：24 = 14 + 11 − 1 ✓。这是两个独立点验证。

**总结论证**：
- Pick 公式 A = I + B/2 − 1 在 CF 的 3 个具体多边形上成立（4 次独立 (A, B, I) 验证：rectangle 原始、shear 后、scale 后、bowtie 失败）。
- summary 的 87 case 在差分意义下与 CF 论断一致（Pick 持续成立 + 变换分类符合 det 准则）。
- 但 summary 不给绝对值，所以 87 case 上的 Pick 公式只能间接确认（pick_holds=True 的标志，没有公式两边的具体数值）。

## 自评

**评分：ARGUMENT（3）**

理由：
- CF 给出 Pick 定理的*应用条件*（lattice + simple）和*变换分类*（det 准则），是机制性陈述；
- 4 个 (A, B, I) 数值点（来自 CF）支持公式；
- summary 87 case 提供差分级一致性（不矛盾，但不能在 case 层独立验证 Pick）；
- 没有 R / T 提供 per-case 的 (A, B, I) 数值或 per-edge 结构，因此论证依赖 CF 文本和小样本数值；
- 达到 ARGUMENT 但缺少 R 提供的"边界 → A、B"的结构推导，所以不到 PROOF。

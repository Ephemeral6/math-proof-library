# Condition R00 — Relation only (boundary_interior)

## 数据分析

`detailed_comparison` 给出每个 case 的 `area_preserved`, `B_change`, `I_change`，以及完整的 `edges_pre/post`（每条边的 from/to/direction/length/gcd/lattice_points_on_edge）。
`structural_comparison` 给出 `pick_holds_pre/post`, `area_from_pick_pre/post = I + B/2 − 1`, `area_from_shoelace_pre/post = A`, `is_lattice`, `is_simple`。

**关键观察 1**：在 87/87 case 上，`area_from_pick == area_from_shoelace`（双侧都成立）。这是 Pick 公式 A = I + B/2 − 1 的直接经验验证。

**关键观察 2**：B 与每条边的 gcd 之间有关系。在每个多边形上，
> B = Σ_e gcd(|Δx_e|, |Δy_e|)

求和遍历所有边。看 `edges_a` 的 `gcd` 字段并求和，恰好等于 summary 里的 `B_a`。例如 rectangle_4x3 边的 gcd 列表是 [4,3,4,3]，sum = 14 = B_a。这说明 **B 完全由边界（每条边的方向向量）决定**——这是 "boundary determines B" 的精确陈述。

**关键观察 3**：4 类变换的不变量行为：
- `translate`: B_change = 0, I_change = 0, area_preserved = True. 平移当然不改变格点结构。
- `shear` (det=1, 整数 entries): B_change = 0, I_change = 0, area_preserved = True. 这是 unimodular 群作用，在整数格上是 bijection。
- `add_vertex` (在边的 interior 格点上加一个新顶点): B_change = 0, I_change = 0, area_preserved = True. 因为新顶点本来就在原边上算作格点，划分不改变 B。
- `scale_non_uniform [sx, sy]`: B_change > 0, I_change > 0, area_preserved = False. A 按 sx·sy 倍放大；B 按"边沿 sx·sy 方向延伸"的 gcd 重新计算；I 按 Pick 反推。

**关键观察 4**：`is_convex`, `is_simple`, `is_lattice` 在所有 87 case 上都是 True。

## 论证尝试

**Shoelace formula** 给出 A 从顶点坐标的计算：
$$A = \frac{1}{2}\left|\sum_{i} (x_i y_{i+1} - x_{i+1} y_i)\right|$$
这是边界数据完全决定 A 的第一个证据（A 只从顶点坐标算，不需要内部信息）。

**B 的边界决定性**：对每条边 (u, v)，边上的格点数 = gcd(|Δx|, |Δy|) + 1。环上每个顶点被两条边共享，去重后
$$B = \sum_e \gcd(|\Delta x_e|, |\Delta y_e|)$$
（这是 `boundary_lattice_sum` 的恒等式，数据中验证。）

**Pick 公式 A = I + B/2 − 1**：
- 由 R 数据：87/87 case 的 `area_from_pick == area_from_shoelace`。这是经验验证。
- 推 I 从 (A, B)：I = A − B/2 + 1，所以**给定 A 和 B（都是边界数据），I 被唯一确定**。
- 这就是"边界决定内部"——在 lattice 简单多边形上，I 的计数完全由顶点坐标决定（通过 Shoelace + gcd 求和）。

**完整证明要求**（数据未直接展示）：
Pick 的标准证明用三角剖分归纳：
- 基础情形：原始三角形（B=3, I=0）面积 = 1/2。
- 归纳步：把多边形沿对角线切成两块，证明 A = A₁ + A₂、B = B₁ + B₂ − 2k（k 是新对角线上的格点数）、I = I₁ + I₂ + (k−1)。代入两侧的 Pick 公式验证总公式。
- R 数据看不到对角线切分，但 `add_vertex` 的"边细分"是相关思想：在边上加格点 → B 不变（合并细分仍是同一边界）+ A 不变 + I 不变。

R 数据没有直接展示三角剖分归纳，但展示了：
- (i) 边界 → A（shoelace）
- (ii) 边界 → B（gcd 求和）
- (iii) Pick 公式在 87 case 上成立

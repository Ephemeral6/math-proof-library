# Condition 00C — counterfactual only (projection)

## 数据分析
summary 之外多了 3 个反事实 case（在 cube × xy 投影上）：

**CF-1 boundary_relaxation（不投影 / 保持 3D）**：  
- 原条件：drop_axis=True，target_dim=2  
- 修改：drop_axis=False，target_dim=3（identity）  
- 原结果：fraction_preserved=43%，4 collisions  
- 修改后：fraction_preserved=100%，0 collisions  
- delta：fraction_change=+0.5714  
- **critical: True** — 维度降低本身是信息损失原因。

**CF-2 operation_perturbation（z 轴 ×0.5 而非丢掉）**：  
- 原条件：orthogonal_projection，axis_scale=0  
- 修改：squash，axis_scale=0.5  
- 原结果：43%，4 collisions  
- 修改后：43%，0 collisions  
- delta：fraction_change=0，collisions -4  
- **critical: True** — 不丢轴只压缩消除 collision，但仍未保持距离。

**CF-3 condition_removal（投影后合并重合点）**：  
- 原条件：preserve_n_points=True (8 points, 8 crossings)  
- 修改：dedupe collided points (4 points, 0 crossings)  
- delta：n_points -4，crossings -8  
- **critical: True** — collision 在数据里编码了 3D 结构。

## 论证尝试

CF 给出三个关键信号：

**(C1) 维度降低 = 距离信息损失的根本原因**（CF-1）：  
保持 3D 时 fraction = 100%；丢掉一个轴 fraction = 43%。  
**Mechanism**: 沿轴分量 ⟨p − q, ê⟩ 一旦被设为 0，距离信息也丢。

**(C2) Collision 来源于 *零化* 而非 *压缩***（CF-2）：  
- 零化（投影）：collisions 4，fraction 43%。  
- 压缩（×0.5）：collisions 0，fraction 43%。  
- 两者距离损失率相同（都没保持原距离），但 collision 只在零化时发生。  
**Mechanism**: collision 需要点对的轴差 ‖Δê‖ 完全为 0；压缩只让 Δê 变小，不为 0。

**(C3) Collision 编码了 3D 结构**（CF-3）：  
- 投影保留 8 点 8 crossings → 能感知到 "原本 8 个点，部分对齐"；  
- Dedupe 后 4 点 0 crossings → 看起来像本来就是 4 点的 2D 对象。  
**Mechanism**: 保留 collision 编码了 "投影前是几个点" 的拓扑信息——dedupe 抹掉这个信息。

但 CF 没有给我：
- 任何具体 case 的 distance matrix 或 collisions list；
- 具体 ratio 公式；
- distances_preserved_count；
- invariants_preserved 字段。

我只能从 3 条 CF 推出三个机制陈述，但无法在 66 个 case 上验证机制是否一致。

## 自评
**评分：PATTERN（2）**

理由：
- CF 给出三条非平凡的机制陈述：维度降低是损失根源 (C1)、零化 vs 压缩的区别 (C2)、collision 编码 3D 信息 (C3)。  
- 这些都是 *正确的洞察*，比 000 多了 mechanism story。  
- 但 CF 只在 1 个代表 case (cube × xy) 上，无法跨 case 验证；没有 invariants_preserved / distance matrix / fraction 数据，构不成 *全局论证*。  
- 比 R00 / 0T0 弱一档（PATTERN vs ARGUMENT）：CF 给出 "what's the cause" 但没给 "how to verify across cases"。

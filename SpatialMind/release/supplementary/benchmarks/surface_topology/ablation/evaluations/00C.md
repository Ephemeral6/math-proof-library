# Condition 00C — C only

## 数据分析
正例 1563 个 (i(α,β) = 1, net_change = 0)。反事实 3 个：

| Strategy                | Modified condition          | net_change | Critical |
|-------------------------|-----------------------------|------------|----------|
| boundary_relaxation     | i(α,β) = 2 (i ≤ 1 → i = 2)  | **−2**     | **TRUE** |
| condition_removal       | β not in descending link    | 0          | FALSE    |
| operation_perturbation  | γ₀ = c0 instead of a₀       | 0          | FALSE    |

## 论证尝试
**反事实告诉我的关键信息**：在已有的几个候选「假设」中——
1. `i(α, β) ≤ 1`
2. β 在 α 的 descending link 中
3. γ₀ = a₀（标准 surgery 曲线选择）

只有 **(1) 是 critical**。(2) 和 (3) 都可以放宽，invariant 仍然保持。

这是一个非常聚焦的「问题简化」信号：我现在知道证明的关键 step 应该围绕 i ≤ 1 vs i ≥ 2 的边界展开，而不是依赖 descending link 或 surgery 曲线选择。

但是我没有任何 R 或 T 数据可以告诉我 *为什么* i ≤ 1 是 critical：
- 我看不到交叉点的几何结构
- 我看不到 surgery 影响哪个区域
- 我只能从 summary delta 推断「i = 2 时 net 减 2」——这告诉我**两个**多余的交叉点被消去了，但消去机制不可见

无法构造证明。但是知道「攻击哪条边界」相比 000 是显著进步。

## 自评
**评分：PATTERN（2）**

理由：识别了 critical condition 的位置（i ≤ 1 是 load-bearing，descending link 和 γ₀ 不是），这是 *structural observation about the problem space*。但没有论证步骤——我只把搜索空间缩小到 i ≤ 1 vs i = 2 的边界，仍然不知道边界两侧机制差在哪里。

# Condition 000 — no R, no T, no C (graph_connectivity)

## 数据分析

100 个 (G, e) pair 的 `summary_delta`：
- `n_components_a`: 取值 0（83 cases）或 +1（17 cases）。
- `is_connected_a`: 取值 `0`（unchanged，83 cases）或 `{"pre": true, "post": false}`（17 cases）。
- `n_edges_a`: 恒 = -1。
- `n_vertices_a`: 恒 = 0。
- `is_connected_b` / `n_components_b`: 恒 = 0（β = 同一个 G）。

`metadata` 给出 graph_idx (0–19), n_vertices (8–12), n_edges_before, deleted_edge, trial — 但**没有**任何关于"这条边在图中起什么作用"的标签。

## 论证尝试

我能看到的：
- 17/100 次删边后 connectivity 从 true 变 false；83/100 不变。
- 同一个 graph_idx 下的 5 次删除有时全保留（如 graph_idx=0 中 5/5 保留），有时混合（具体哪个图我看不见结构）。
- 17 个 connectivity-loss case 和它们的 deleted_edge 之间有什么共性？我不知道——我只看到一个 (u, v) pair，看不到这条边在哪个图里、邻居有谁。

我能猜测的机制："存在某个图论性质 P(G, e)，使得 connectivity_lost ⇔ P(G, e) 成立。" 但我没有能区分两类的数据。删除的边 (u, v) 的端点编号 / graph_idx / n_vertices 之间没有可观察的差异（所有 cases 的 n_vertices 都在 8–12，连接 lost 和保持的两组在这些字段上分布看不出区别）。

## 自评

**评分：PATTERN−（1）**

理由：观察到 17/100 的 connectivity-loss frequency，是*真实的*结构现象（deleted_edge 的某个属性决定 outcome），但我无法从 summary 数据中提取这个属性是什么。我有 outcome 但没有机制。
